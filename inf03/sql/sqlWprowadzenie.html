Witam w Wprowadzeniu do SQL! Tutaj są materiały, które pomogą się przygotować z SQL!

SQL (ang. Structured Query Language) jest językiem zapytań, przy pomocy którego komunikujemy się z bazą danych
(wydajemy polecenia, które pozwalają zarządzać znajdującymi się tam danymi).


Kwerendy SQL

Kwerendy w języku SQL nazywane są również zapytaniami - służą one do uzyskiwan określonych rekordów z tabeli, dodawania nowych, aktualizacji istniejących bądź usuwania określonego rekordu. Zapytania to podstawowa forma komunikacji z bazą danych. Będziemy odpytywać naszą bazę, by zwróciła nam rekordy, które spełniają określony warunek lub dodata nowy rekord do bazy danych.


Kwerenda SELECT

Kwerenda SELECT to zapytanie wybierające rekordy z tabeli. Za jej pomocą możemy wyciągnąć z tabeli określone przez nas dane.
Kwerenda SELECT wybiera rekordy o określonych kolumnach z określonej tabeli. Jej składnia wygląda tak:


    SELECT nazwa_kolumny FROM nazwa_tabeli;

Kwerenda SELECT - warunki

Możemy wybierać rekordy wyłącznie spełniające jakiś warunek - na przykład gdzie wartość kolumny pensja jest większa od 3500.
Do ustawienia warunku w zapytaniu SELECT, użyjemy słówka kluczowego WHERE.
Przykładowo: wyciągnijmy wszystkie rekordy, które mają wartość pensji większą niż 3500.


    SELECT * FROM tabela WHERE pensja > 3500;

Kwerenda SELECT - warunki

Możemy również łączyć warunki - kolejne warunki będziemy oddzielać słowem AND. Wtedy, by rekord został wybrany, muszą zostać spełnione wszystkie warunki.
Możemy skorzystać także z alternatywy - warunki oddzielamy słowem OR. Wtedy rekordy zostaną wybrane, gdy przynajmniej jeden warunek będzie spełniony.


Wybieranie rekordów, gdzie podana jest część tekstu

Aby otrzymać rekord, którego wartość kolumny rozpoczyna się bądź kończy na danej literze lub która zawiera określone stowo, musimy skorzystać z operatora LIKE w warunku WHERE.
- WHERE kolumna LIKE '%a' - zwróci rekordy, których wartość kolumny zaczyna się od a
- WHERE kolumna LIKE 'a%' - zwróci rekordy, których wartość kolumny kończy się na a
- WHERE kolumna LIKE '%war%' - zwróci rekordy, których wartość kolumny posiada słowo "war".


Sumowanie wartości z kolumn - Funkcja SUM

Aby zapytanie SELECT zwróciło sumę wartości z danej kolumny (na przykład sumę wszystkich ilości towarów w zleceniach), należy po słowie SELECT wstawić funkcję SUM(kolumna).
Przykładowo: zesumujmy wszystkie wartości pensja dla tabeli pracownicy.


    SELECT SUM(pensja) FROM pracownicy;

Maksymalna/minimalna wartość kolumny - Funkcja MAX/MIN

Aby zapytanie SELECT zwróciło maksymalną lub minimalną wartość z danej kolumny należy po słowie SELECT wstawić funkcję MAX(kolumna) lub MIN(kolumna).
Przykładowo: wybierz staż pracy pracownika, który jest najdłuższy z całej tabeli pracownicy.


    SELECT MAX(staz_pracy) FROM pracownicy;
    // dla maksymalnej (patrz wyżej), oraz minimalnej (patrz niżej)
    SELECT MIN(staz_pracy) FROM pracownicy;

Średnia z wartości kolumny - Funkcja AVG

Ostatnią z funkcji dotyczącą zapytania SELECT jest AVG, które zwracać będzie średnią wartość z danej kolumny.
Zapis jest identyczny jak we wcześniejszych dwóch przypadkach - stawiamy funkcję AVG(kolumna) po słowie SELECT.


    SELECT AVG(kolumna) FROM tabela;

Ograniczenie ilości wyników

Czasem będziemy chcieli uzyskać tylko 5 rekordów, które spełniają nasze wymagania.
Do tego będzie służyć nam funkcja LIMIT, którą będziemy umieszczać na samym końcu zapytania SELECT.
Po słowie LIMIT podajemy liczbę rekordów, którą chcemy otrzymać w wyniku.


    SELECT * FROM pracownicy WHERE pensja > 3500 LIMIT 5 
    // zwróci wyłącznie 5 pracowników spełniających warunek.

Kwerenda INSERT

Zapytanie INSERT INTO będzie wstawiało nowy rekord do tabeli. Jego składnia będzie wyglądać tak:


    INSERT INTO tabela VALUES (wartość dla kolumny 1, wartość dla kolumny 2...);

To zapytanie można przetłumaczyć na język polski jako włóż do tabeli o nazwie tabela dane wartości.


Kwerenda UPDATE

Kwerenda UPDATE działać będzie wyłącznie na rekordy, które już istnieją. Będzie ona aktualizować określony rekord lub rekordy i nadawać im nowe wartości.


    UPDATE tabela SET kolumna=wartość WHERE kolumna2=wartość2;

Zwróć uwagę na warunek WHERE w zapytaniu. Będzie on działać identycznie jak w przypadku zapytania SELECT.


Kwerenda DELETE

Kwerenda DELETE będzie usuwać dane rekordy, które spełniają określony warunek. Składnia:


    DELETE FROM tabela WHERE kolumna=wartość;

!!! Warunek WHERE jest obowiązkowy - bez niego usunięte zostaną wszystkie rekordy z tabeli !!!

Z tego zadania nam zostało jeszcze dodać nazwę tabeli z której chcemy wyświetlić te kolumny, zatem:

Drugie zadanie:

Mamy przeliczyć ilość lotów, które się odbywają pomiędzy 10:00:00 a 10:59:00.

Najpierw weźmy nasze wyświetlanie, czyli SELECT


    SELECT nazwa_kolumny FROM nazwa_tabeli;

Teraz skoro chcemy przeliczyć to wykorzystujemy naszą funkcję COUNT do przeliczenia lotów: 


    SELECT COUNT(id) FROM nazwa_tabeli;

Teraz napiszmy z jakiej tabeli to pobieramy te dane: 


    SELECT COUNT(id) FROM przyloty;

W tym momencie nam zliczy wszystkie przyloty, lecz my tego nie chcemy.
Mamy w poleceniu zadania sprecyzowane które przyloty mają być przeliczone, czyli te od 10:00:00 do 10:59:00, zatem zastosujemy WHERE


    SELECT COUNT(id) FROM przyloty WHERE czas >= "10:00:00"

Jak można zauważyć, wykorzystaliśmy tutaj WHERE, który nam sprecyzował wynik do tylko przylotów od godziny 10:00:00 w górę przez znaki ">=", czyli większy/lub równy.
Oznacza to, że teraz to się zliczy wszystko od godziny 10 w górę, lecz jeszcze nie mamy granicy górnej, czyli do godziny 10:59:00. Do tego użyjemy AND


    SELECT COUNT(id) FROM przyloty WHERE czas >= "10:00:00" AND czas <= "10:59:00";

Musimy teraz zaktualizować status lotu na "opoznienie 20 min" dla numeru rejsu "LX5673"

    UPDATE nazwa_tabeli SET nazwa_kolumny = wartość WHERE nazwa_kolumn1 = sprecyzowana_wartość

Zatem najpierw podajmy nazwę naszej tabeli: 


    UPDATE odloty SET nazwa_kolumny = wartość WHERE nazwa_kolumn1 = sprecyzowana_wartość;

Podajmy teraz nazwę kolumny, w której będzie zmieniana wartość, czyli dla status_lotu, gdyż zmieniamy status: 


    UPDATE odloty SET `status_lotu` = wartość WHERE nazwa_kolumn1 = sprecyzowana_wartość;

Teraz podajemy jaka wartość ma się wyświetlić w kolumnie status_lotu, czyli tekst "opoznienie 20 min": 


    UPDATE odloty SET `status_lotu` = "opoznienie 20 min" WHERE nazwa_kolumn1 = sprecyzowana_wartość;

    UPDATE odloty SET `status_lotu` = "opoznienie 20 min" WHERE nr_rejsu = sprecyzowana_wartość;
